# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

파워리프팅 프로그램(트레이닝 프로그램)을 제작·관리하기 위한 도구.

## Current State

스캐폴딩 단계입니다. 폴더 골격만 잡혀 있고 프론트엔드/백엔드 스택은 아직 결정되지 않았습니다. 빌드·테스트·실행 명령은 스택이 정해지는 대로 이 문서에 추가하세요.

## Repository Layout

- `frontend/` — 프론트엔드 앱 (스택 미정)
- `backend/` — 백엔드 서비스 (스택 미정)
- `plan/` — 코드 개발에 필요한 계획 문서 (작업 단위 설계, 의사결정 기록 등)
  - `plan/claude/` — Claude 가 작성하는 제안서·검토 노트·작업 메모
- `docs/` — 그 외 참고 문서 (도메인 지식, 외부 자료 정리 등)

`plan/` 과 `docs/` 의 구분: 구현 직전의 "무엇을 어떻게 만들지" 는 `plan/`, 구현과 직접 묶이지 않는 배경 지식·자료는 `docs/`.

## Conventions

### `plan/` 영역 분리

- `plan/` 루트는 **사용자 전용 영역**입니다. 사용자가 Claude 에게 요청할 내용을 기록합니다. Claude 는 루트의 파일을 만들거나 수정하지 않습니다 (사용자가 명시적으로 지시할 때만 예외). 읽는 것은 자유.
- `plan/claude/` 는 **Claude 작성 영역**입니다. 제안서, 검토 노트, 탐색 결과 등을 자유롭게 쓰고 정리합니다.

### Claude 의 작업 기록은 git 에 직접

별도의 plan/요약 문서를 레포에 두지 않습니다. 의도와 결과는 **커밋 메시지 / PR description** 으로 남깁니다.

작업 단위 기준:

- **작은 건 → `main` 직커밋**: 단일 폴더 안 변경 + 대략 50줄 미만 + 동작 변경 없음 (문서/컨벤션/스캐폴딩/타입 수정 등)
- **큰 건 → 브랜치 + PR**: 여러 폴더에 걸친 변경, 새 기능/마이그레이션/리팩토링, 동작 변경이 있거나 리뷰가 필요한 것
- 애매하면 PR 쪽으로

커밋/PR 본문은 "무엇을 왜 바꿨는가" 가 한 번에 보이게 작성하세요.

### 기타

- 도메인 자료(예: 프로그램 공식, 1RM 계산식, 참고 논문 정리)는 `docs/` 에 둡니다.
- 스택을 정해 init 한 뒤에는 해당 폴더 안에 별도의 `CLAUDE.md` 를 두거나 이 루트 문서에 명령을 추가하세요.
