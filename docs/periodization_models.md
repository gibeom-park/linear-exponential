# 주기화 프로그램 (Periodization Models)

코치 모드에서 사용자가 선택할 수 있는 4종 주기화 프로그램의 정의/특성/적합 대상.

- **참조 위치**: [`plan/claude/domain_model.md`](../plan/claude/domain_model.md) §2
- **default 프로그램**: Linear
- **프롬프트 매핑**: 각 프로그램은 `backend/prompts/coach_{program_type}.md` 와 1:1 대응

각 프로그램의 **UI 카드 한 줄 설명** 은 코치 모드 진입 시 카드 컴포넌트에서 그대로 사용한다.

---

## 1. Linear Periodization (선형) — default

**UI 한 줄 설명**: "주차가 갈수록 무게는 ↑, 볼륨은 ↓ 단순하고 예측 가능."

### 작동 방식
- 블럭 시작 주차에서는 고볼륨·저강도 (예: 5×5 @70%)
- 주차가 진행될수록 볼륨 ↓, 강도 ↑ (예: 6주차 3×3 @90%)
- 한 주기 끝에 1RM 시도 또는 테스트

### 적합 대상
- 입문 ~ 중급 (linear progression 이 아직 작동하는 단계)
- 시합 일정이 비교적 멀거나 없는 시기
- 단순한 진행을 원하는 경우

### 장단점
- ✅ 구조가 명확. LLM 이 프로그래밍하기 가장 쉬움
- ❌ 장기간 사용 시 정체. 자극이 단조로움
- ❌ 한 주기 내 hypertrophy / strength 둘 중 하나만 강조됨

---

## 2. DUP (Daily Undulating Periodization, 일일 변동 주기화)

**UI 한 줄 설명**: "한 주 안에서 매 세션 강도/볼륨을 다르게. 다양한 자극."

### 작동 방식
- 같은 운동을 한 주 내에서 서로 다른 영역으로 훈련
- 예: 월 5×5 @75% (strength), 수 3×10 @65% (hypertrophy), 금 3×3 @90% (peaking)
- 매 세션마다 자극이 바뀌어 회복 + 적응 사이클 단축

### 적합 대상
- 중급 이상 (단일 자극으로는 정체가 오는 단계)
- 주 3~5일 훈련 가능한 사람
- 정체기 돌파 / 신체 적응 다변화 목적

### 장단점
- ✅ 짧은 사이클 안에 strength + hypertrophy 동시 자극
- ❌ 회복 관리 난이도 ↑ (RPE 모니터링 필수)
- ❌ 세션별 특성이 달라 fatigue 누적 추적이 까다로움

---

## 3. Block Periodization (블럭 주기화)

**UI 한 줄 설명**: "3~4주 단위 블럭으로 한 가지 목표에 집중. 시합 사이클에 최적."

### 작동 방식
- 블럭마다 단일 목표:
  1. **Accumulation block** (볼륨/근비대 축적, 3~4주)
  2. **Intensification block** (강도 ↑ 근력 전환, 3~4주)
  3. **Realization / Peaking block** (피크 + 시합 준비, 1~2주)
- 누적된 적응을 마지막 블럭에서 발현시키는 구조

### 적합 대상
- 중상급 ~ 엘리트
- 시합 일정이 명확하고 사이클을 역산할 수 있는 시기
- 8주 이상의 장기 계획이 가능한 경우

### 장단점
- ✅ 시합 피크 타이밍을 정밀하게 맞출 수 있음
- ✅ 한 블럭 안에서 한 가지 적응에만 집중 → 효과적
- ❌ 한 블럭 내 단조로운 자극 (입문자에게는 부적합)
- ❌ 블럭 간 전환에서 손실되는 적응 존재

---

## 4. Conjugate (Westside Barbell)

**UI 한 줄 설명**: "Max-Effort + Dynamic-Effort 분리. 변형 운동 풀이 커야 함."

### 작동 방식
- 한 주에 두 종류의 day 를 배치:
  - **Max Effort (ME) day**: 변형 운동으로 1~3RM 시도 (매주 변형 운동 교체)
  - **Dynamic Effort (DE) day**: 60~70% 무게에 밴드/체인을 더해 폭발력 훈련
- 보조 운동(accessory) 비중이 큼 (약점 부위 집중 강화)

### 적합 대상
- 상급자 (최소 2~3년 이상의 일관된 훈련 경험)
- 다양한 변형 운동에 접근 가능한 환경 (장비 / 보조 도구)
- 정체 돌파 + 약점 보강이 우선순위

### 장단점
- ✅ 약점 부위 타게팅에 가장 강력
- ✅ ME 운동 교체로 정체가 거의 없음
- ❌ 장비/공간 요구가 큼 (밴드, 체인, 박스, 보드 등)
- ❌ RAW 리프터에게 ME 운동 선택이 까다로움 (gear 전제 프로그램에서 출발)

## 비교 요약

| 항목 | Linear | DUP | Block | Conjugate |
|---|---|---|---|---|
| 추천 레벨 | 입문~중급 | 중급+ | 중상급~엘리트 | 상급 |
| 한 주기 길이 | 4~8주 | 4~6주 | 8~12주 | 무제한(연중) |
| 주당 자극 다양성 | 낮음 | 높음 | 낮음 | 매우 높음 |
| 회복 관리 난이도 | 낮음 | 높음 | 중간 | 높음 |
| 시합 피크 적합도 | 중간 | 낮음 | 매우 높음 | 낮음 |
| 변형 운동 풀 필요량 | 작음 | 작음 | 중간 | 매우 큼 |
| LLM 프롬프팅 난이도 | 쉬움 | 중간 | 중간 | 어려움 |

## 프롬프트 작성 가이드

각 프로그램별 LLM 프롬프트(`backend/prompts/coach_{program_type}.md`) 작성 시 공통 입력:
- 사용자 1RM (SBD 각각, kg)
- 블럭 길이 (`weeks`)
- 주당 일수 (`days_per_week`)
- 메인 데드리프트 스탠스 (conv / sumo)
- 사용자 약점 메모 (선택)

프로그램별 특화 입력:
- **Linear**: 목표 1RM 증가 폭 (예: +5kg)
- **DUP**: 주당 strength/hypertrophy/peaking 비율
- **Block**: 시합일 (있다면) → 역산해 블럭 길이 자동 분배
- **Conjugate**: ME 변형 풀, DE 밴드/체인 가용 여부

출력 JSON 스키마는 모두 동일: `program_sets` 테이블 적재 가능한 형태 (block_id / week_no / day_no / exercise_id / set_no / planned_reps / planned_weight_kg).
